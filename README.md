# 📝 Animation & Simulation of Operating System Semaphores

[Simulation Video](https://youtu.be/EUjkejbAMOc)

### 🔍 Overview
This collaborative system programming project focused on bridging theoretical operating system concepts with real-world technical execution. Working in a group of 4 members, we developed an interactive animation and simulation program designed to visually explain and execute the mechanics of Semaphores—a fundamental OS synchronization tool used to manage concurrent processes and prevent race conditions within a shared memory space.

### 🛠️ Key Takeaways & Implementation
- Translating Complex OS Concepts: Semaphores (wait/P and signal/V operations) can be highly abstract. Our team successfully mapped these low-level synchronization principles onto a concrete, real-world application scenario, making concurrent thread scheduling easy to visualize and comprehend.

- Concurrent Programming & Simulation Coding: I gained practical experience handling multi-threaded or multi-process execution environments. Building the simulation backend required managing critical sections, preventing deadlocks, and demonstrating how a semaphore controls access to finite resources under high-traffic conditions.

- Interactive Visualization: Beyond writing the logic, we engineered a clean graphical interface or programmatic animation sequence. This UI visually tracked thread states (e.g., Blocked, Waiting, Executing), offering a real-time view of data flow and locking mechanisms during execution.

- Group Synergy & Software Integration: Coordinating with team members to synchronize the simulation backend logic with the frontend visual layers sharpened our unified system design and collaborative coding workflows.

### 💡 Conclusion
This project deepened my command over low-level system design, process synchronization, and concurrency control models. Transitioning from abstract algorithm definitions to building an interactive, working visual simulator demonstrated the critical importance of robust mutual exclusion and resource allocation in production-grade software architectures.
